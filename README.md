# ☕ Myko Coffee — Full-Stack Інтернет-Кав'ярня

**Myko Coffee** — це сучасний веб-додаток для кав'ярні, який дозволяє гостям вибирати улюблені напої, додавати їх у кошик та оформлювати швидкі передзамовлення на сьогодні з автоматичною валідацією робочих годин. Для бариста передбачена інтерактивна адмін-панель керування замовленнями з оновленням статусів у реальному часі.

---

## 🚀 Основні можливості

### 📱 Для гостей кав'ярні:
- **Інтерактивне меню:** Великий вибір кави та авторських напоїв з можливістю детального налаштування (розмір порції, додаткові опції, сиропи).
- **Динамічний кошик:** Зручне додавання, видалення та зміна кількості напоїв з автоматичним підрахунком вартості.
- **Розумне передзамовлення на сьогодні:**
  - Оформити замовлення можна лише на поточну добу.
  - Робочі години кав'ярні: **з 08:00 до 22:00**.
  - Форма автоматично блокується поза робочим часом із виведенням повідомлення.
  - При виборі часу пропонується найближчий доступний слот (через 15 хвилин від поточного часу) або можливість вказати свій час (валідується, щоб обраний час не був у минулому).
- **Адаптивний інтерфейс:** Сайт виглядає преміально на будь-яких пристроях (смартфони, планшети, ПК) завдяки детально пропрацьованій CSS-сітці та анімаціям.

### ☕ Для бариста (Адмін-панель):
- **Моніторинг замовлень:** Отримання нових замовлень з бази даних кожні 10 секунд без перезавантаження сторінки.
- **Керування статусами:** Швидка зміна статусів замовлення у реальному часі: `Нове` ➔ `Готується` ➔ `Готово` ➔ `Завершено` / `Скасовано`.
- **Жива статистика:** Зручні таби-фільтри з кольоровими індикаторами кількості замовлень за кожним статусом.
- **Повна інформація:** Детальний список вибраних напоїв з об'ємом, ціною та контактами клієнта для швидкого зв'язку.

## 🛠️ Технологічний стек

- **Frontend:**
  - Чистий **HTML5** (семантична верстка)
  - **Vanilla CSS3** (CSS-змінні, Flexbox, Grid, адаптивні медіа-запити, плавні мікроанімації)
  - **ES6+ JavaScript** (асинхронні запити `fetch`, динамічний рендеринг DOM, маски вводу)
- **Backend:**
  - **Node.js** та веб-фреймворк **Express.js**
  - **SQLite** (`sqlite3`) — швидка локальна база даних
- **Тестування:**
  - Автоматизовані unit-тести на базі **Jest** (покривають бізнес-логіку підрахунку цін, валідації телефонів та часу)

## 📂 Структура проєкту

```text
coffeeshop/
├── database.sqlite       # Локальна база даних SQLite (генерується при старті)
├── package.json          # Залежності проєкту та npm-скрипти
├── server.js             # Express.js бекенд сервер (налаштування БД, API маршрути)
├── index.html            # Головна сторінка сайту (меню, кошик, форма передзамовлення)
├── app.js                # Фронтенд-логіка клієнтської частини сайту
├── admin.html            # Веб-інтерфейс адмін-панелі бариста
├── admin.js              # Фронтенд-логіка для керування замовленнями в адмінці
├── styles.css            # Загальна таблиця стилів сайту та адмін-панелі
├── app.test.js           # Набір Jest-тестів для перевірки логіки додатку
└── README.md             # Документація проєкту
```

## 🏁 Як запустити проєкт локально

### 1. Клонування репозиторію та інсталяція залежностей
```bash
# Клонуйте репозиторій
git clone https://github.com/canyoud1e/coffeeshop.git
cd coffeeshop

# Встановіть необхідні NPM-пакети
npm install
```

### 2. Запуск локального сервера
```bash
npm start
```
*Після запуску ви побачите повідомлення про успішне підключення до бази даних SQLite та номер порту.*

### 3. Перехід на сторінки у браузері
- Головна сторінка для гостей: **[http://localhost:3000](http://localhost:3000)**
- Панель керування замовленнями бариста: **[http://localhost:3000/admin.html](http://localhost:3000/admin.html)**

### 4. Запуск автоматичних тестів
Якщо ви вносите зміни до логіки додавання замовлень або підрахунку цін, перевірити коректність роботи можна командою:
```bash
npm test
```

## 📄 Схема бази даних
Додаток використовує дві зв'язані таблиці в SQLite:
1. `orders` — загальна інформація про замовлення (ім'я, телефон, дата, час та статус: `new`, `preparing`, `ready`, `completed`, `cancelled`).
2. `order_items` — конкретні позиції у кожному замовленні (назва напою, об'єм, кількість та підсумкова ціна).
